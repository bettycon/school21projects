##### 2. Написать при помощи библиотеки Micrometer сборщики следующих метрик приложения:

- количество отправленных сообщений в rabbitmq;
- количество обработанных сообщений в rabbitmq;
- количество бронирований;
- количество полученных запросов на gateway;
- количество полученных запросов на авторизацию пользователей.

Для использования библиотеки  Micrometer (библиотека для сбора метрик в Java-приложениях) необходимо добавить зависимости в конфигурационные файлы  проектов Maven (Java). В каждом из микросервисов нашего проекта существует файл pom.xml, в который необходимо добавить зависимости Micrometer  и Spring Boot Actuator для сбора метрик с наших эндпоинтов:

![](./images/3.png) </br>


Для создания сборщика метрик, указанных в заданиия нам необходимо реализовать Spring Boot компонент, предназначенный для сбора и регистрации метрик приложения. Он служит централизованной системой мониторинга требуемых событий и операций:

![](./images/4.png)

В каждом из необходимых сервисов создаем ссылку на общий компонент сбора метрик:

@Autowired

private MetricsCollector metricsCollector;

Затем в каждом из микросервисов, где собираем метрики, создаём счетчик, который инкрементится при вызове каждого из необхоимых нам методов:

- Для мониторинга количества  обработанных сообщений в rabbitmq. Редактируем файл booking-service/src/main/java/com/s21/devops/sample/bookingservice/Statistics/QueueProducer.java.
Добавляем метод инкремент для увеличения счетчика событий при  обработке сообщений.

![](./images/5.png)


- Для мониторинга количества отправленных сообщений в rabbitmq. Редактируем файл report-service/src/main/java/com/s21/devops/sample/reportservice/Statistics/QueueConsumer.java
Добавляем метод инкремент для увеличения счетчика событий при  отправке  сообщений.

![](./images/6.png)


- Для мониторинга количества бронирований. Редактируем файл booking-service/src/main/java/com/s21/devops/sample/bookingservice/Service/BookingServiceImplementation.java
Добавляем метод инкремент для увеличения счетчика событий при  бронировании.

![](./images/7.png)

![](./images/8.png)


- Для мониторинга количества полученных запросов на gateway. Редактируем файл gateway-service/src/main/java/com/s21/devops/sample/gatewayservice/Controller/GatewayController.java
Добавляем метод инкремент для увеличения счетчика событий при  каждом обращении на шлюз.

![](./images/9.png)

- Для мониторинга количества  полученных запросов на авторизацию пользователей. Редактируем файл session-service/src/main/java/com/s21/devops/sample/sessionservice/Service/SessionServiceImplementation.java
Добавляем метод инкремент для увеличения счетчика событий при  получении запросов на авторизацию.

![](./images/10.png)



Для  доступа к мониторингу  всех эндпоинтов  и включения поддержки Prometheus необходиммо также отредактировать файл конфигурации для Spring Boot приложений application.properties. Он также есть в каждом из микросервисов исходного приложения.

![](./images/11.png)

Добавляем:

 " management.endpoints.web.exposure.include=* " // - Открывает все доступные эндпоинты Actuator через HTTP

 " management.endpoint.prometheus.enabled=true " // - Включает специальный эндпоинт actuator/prometheus для экспорта  метрик в Prometheus


После редактирования конфигурационных файлов Spring и внесения изменений в работу микросервисов, нам необходимо пересобрать докер-файлы измененных микросервисов и залить их снова на докер-хаб, чтобы наш docker-compose.yml осуществлял корректную сборку.
